Migrando a SQLite 🚀

Taller POS_TAP

Pasando de archivos de texto a una verdadera Arquitectura Profesional.

¿Por qué SQLite?

  • Dejamos de depender de diccionarios gigantes cargados en RAM.
  • Garantizamos la Integridad de los Datos (Relaciones entre tablas).
  • Podemos hacer Transacciones para evitar errores "a medias".

1. La Conexión Inteligente

En nuestro data_manager.py no solo abrimos el `.db`, sino que lo configuramos como un profesional:


def _get_conn(self) -> sqlite3.Connection:
    conn = sqlite3.connect(self.db_path)
    
    # ¡El truco de oro! Retorna filas como diccionarios
    conn.row_factory = sqlite3.Row        
    
    # Activa validación de relaciones
    conn.execute("PRAGMA foreign_keys = ON")
    return conn

2. Todo está Relacionado

En lugar de guardar un ticket dentro de un JSON, ahora tenemos tablas conectadas por IDs:


CREATE TABLE ventas (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    fecha TEXT,
    total REAL
);

CREATE TABLE venta_detalle (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    venta_id  INTEGER REFERENCES ventas(id) ON DELETE CASCADE,
    producto  TEXT,
    cantidad  INTEGER
);

3. Transacciones Atómicas ⚛️

Cuando cobramos, ocurren 3 cosas. El `with` asegura que si una falla, se cancelan las demás.


def registrar_venta(self, carrito, total):
    with self._get_conn() as conn:
        
        # 1. Creamos el recibo (venta general)
        cursor = conn.execute("INSERT INTO ventas ...", (fecha, hora, total))
        venta_id = cursor.lastrowid
        
        for prod, cant in carrito.items():
            # 2. Insertamos la lista de cosas vendidas
            conn.execute("INSERT INTO venta_detalle (venta_id...) VALUES...", 
                         (venta_id, prod, cant))
                         
            # 3. Y le restamos al INVENTARIO todo al mismo tiempo
            conn.execute("UPDATE productos SET stock = stock - ? ...", 
                         (cant, prod))
